# Equipo

Empresa **bootstrapped**, dos fundadores, sin inversión externa.

- **[[Xavi (CEO)]]** — perfil comercial, acceso institucional, viene del sector talento/universidades.
- **[[Agu (CTO)]]** — perfil técnico fuerte, viene de Turijobs (lado job-board).

> [!note] Por qué este equipo
> El problema requiere dos cosas que rara vez van juntas: **acceso institucional** + **capacidad técnica para construir infraestructura compleja**. El CEO sabe cómo compran las universidades; el CTO tiene que construir un sistema que dure décadas, no una app que dure dos años.

## Qué se busca en el CTO (del briefing)
Sistemas backend complejos (no solo APIs) · comodidad con event-driven y state machines · criterio técnico propio (al leer [[ADR-001 - Relational + Event Log]] tiene algo que decir) · tolerancia a la ambigüedad de fase temprana · interés genuino en el problema.
